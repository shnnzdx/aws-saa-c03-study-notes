# Amazon S3

## Overview

Amazon S3 is object storage designed for durability, scalability, and a wide spectrum of access patterns.

## When to use

- Store static website assets and backups
- Archive logs and media assets
- Host large datasets for analytics

## Common decision logic

- Use S3 Standard for frequently accessed objects.
- Use S3 Standard-IA or Glacier for long-term retention and infrequent access.
- Use Intelligent-Tiering when access patterns are unknown.

## Design implications

- Protect sensitive data with encryption at rest and in transit.
- Use lifecycle policies to reduce storage costs over time.
- Use bucket policies and IAM roles to enforce least privilege.

## Practical points

- S3 is not a file system; it is optimized for object-level access.
- Choosing the right storage class can have a large cost impact.
- S3 integrates well with CloudFront for global content distribution.
