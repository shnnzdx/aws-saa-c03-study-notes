# AWS SAA-C03 Exam Overview

## Purpose

This document organizes the exam domain areas and study priorities for AWS SAA-C03. It is intended to help me focus on architecture decisions rather than memorizing details.

## Key domains

- **Design resilient architectures**
- **Define performant architectures**
- **Specify secure applications and architectures**
- **Design cost-optimized architectures**
- **Define operationally excellent architectures**

## Study focus

- Understand why AWS services are chosen for a use case
- Learn how components interact in a scalable application
- Practice comparing alternatives using architecture trade-offs
- Focus on security controls, network design, and data services

## Practical guidance

- Review common AWS service families: compute, storage, database, networking, security, monitoring
- Map real requirements to AWS patterns: static website, three-tier web app, serverless workflow, hybrid connectivity
- Practice explaining decisions in terms of availability, performance, security, and cost

## When to use

- When evaluating AWS solutions for interviews or cloud design discussions
- When reinforcing the connection between concepts and real-world architecture
- When preparing portfolio materials that show practical architecture reasoning
