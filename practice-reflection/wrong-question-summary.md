# Practice Reflection: Wrong Question Summary

## Purpose

This note captures lessons learned from practice scenarios and design decisions that need review.

## Observations

- Focus on architecture decisions, not memorization of specific steps.
- Clarify why a particular service is a better fit for the use case.
- Review security requirements and how they change service choices.

## Common learning points

- I tend to choose the most familiar service instead of the best-fit option.
- I need to practice identifying performance and availability trade-offs more consistently.
- I should verify whether a design requires private connectivity or public access.

## Action items

- Revisit use cases for S3 storage classes and database choices.
- Practice explaining why a solution uses managed service endpoints or VPC endpoints.
- Build a checklist for security, scaling, and cost trade-offs in architecture reviews.
